# Semester Scratch Course (4-5 Months)

**Target Audience:** School children ages 8-12
**Duration:** 4-5 months (16-20 weeks)
**Frequency:** 1-2 hours per week
**Level:** Beginner → Intermediate

---

## Course Overview

This streamlined 4-5 month course covers the essential Scratch programming concepts and builds a strong foundation in beginner and intermediate skills. Students will create interactive projects and games while learning key programming principles.

---

## Learning Objectives

By the end of this semester, students will be able to:
- Navigate and understand the Scratch interface
- Create interactive projects using sprites and scripts
- Implement loops, conditionals, and basic logic
- Use variables to manage game state
- Design and create simple games
- Share and present their work

---

## Unit 1: Scratch Fundamentals (Weeks 1-4)

### Week 1-2: Introduction to Scratch

**Learning Objectives:**
- Set up Scratch account
- Navigate the Scratch workspace
- Understand basic concepts (sprites, stage, blocks)
- Create first interactive program

**Topics:**
- Scratch interface overview
- Sprite and stage concepts
- Motion and appearance blocks
- Event blocks (when clicked, when key pressed)

**Activities:**
- Explore Scratch workspace
- Create moving sprite
- Make sprite respond to keyboard

**Project:** "Moving Character"
- Use keyboard to move sprite in multiple directions
- Include sound effect on movement

**Assessment:** Sprite responds correctly to user input

---

### Week 3-4: Events and Basic Interaction

**Learning Objectives:**
- Understand event-driven programming
- Use multiple event triggers
- Chain blocks together logically

**Topics:**
- Event blocks deep dive
- Combining multiple scripts
- Sound and appearance changes
- Debugging simple programs

**Activities:**
- Create sprite with multiple responses
- Add sound effects
- Create visual feedback for user actions

**Project:** "Interactive Character"
- 3+ different keyboard responses
- Sound effects
- Costume changes
- Visual feedback on interactions

**Assessment:** Fully interactive character with multiple responses

---

## Unit 2: Loops and Sequences (Weeks 5-8)

### Week 5-6: Introduction to Loops

**Learning Objectives:**
- Understand the concept of repetition
- Use "repeat" and "forever" blocks
- Recognize when loops are appropriate
- Create animations using loops

**Topics:**
- Repeat block (fixed iterations)
- Forever block (infinite loops)
- Loop efficiency and applications
- Common loop patterns

**Activities:**
- Create sprite animations
- Make moving backgrounds
- Build automated sequences

**Project:** "Animated Scene"
- Create at least 2 animated sprites
- Include background motion
- Use loops for efficiency

**Assessment:** Working animations created with loops

---

### Week 7-8: Multiple Sprites and Coordination

**Learning Objectives:**
- Work effectively with multiple sprites
- Control individual sprite behaviors
- Create basic interactions between sprites
- Coordinate multiple simultaneous actions

**Topics:**
- Managing multiple sprites
- Broadcast basics
- Collision detection introduction
- Scene/screen management

**Activities:**
- Create multi-sprite scenes
- Implement simple interactions
- Create sequential scenes

**Project:** "Interactive Scene"
- 4+ sprites with individual behaviors
- Sprites interact when clicked
- Multiple scenes (at least 2)
- Scene transitions

**Assessment:** Multi-sprite scene with working interactions

---

## Unit 3: Conditionals and Logic (Weeks 9-12)

### Week 9-10: If/Else Statements

**Learning Objectives:**
- Understand conditional logic
- Use if and if/else blocks
- Apply comparison operators
- Create decision-based programs

**Topics:**
- If block functionality
- If/else block structure
- Comparison operators (>, <, =, etc.)
- Logical conditions in games

**Activities:**
- Create programs with decision points
- Implement scoring logic
- Add game rules

**Project:** "Simple Game (Part 1)"
- Use conditionals for game rules
- Implement basic scoring
- Add win/lose conditions

**Assessment:** Program with working conditional logic

---

### Week 11-12: Variables and Game State

**Learning Objectives:**
- Understand variables and their purpose
- Create and modify variables
- Use variables in conditionals
- Manage game state with variables

**Topics:**
- Creating variables
- Setting and changing values
- Using variables in conditions
- Score and lives systems
- Variable naming conventions

**Activities:**
- Create score tracker
- Implement lives system
- Build game with variable-based logic

**Project:** "Complete Game"
- Incorporate variables for score/lives
- Use conditionals based on variables
- Proper win/lose conditions
- 3+ sprites
- Playable and enjoyable

**Assessment:** Fully functional game with variables and conditionals

---

## Unit 4: Game Design and Capstone (Weeks 13-16/20)

### Week 13-14: Game Design Principles

**Learning Objectives:**
- Understand core game design concepts
- Plan game mechanics
- Design user-friendly interfaces
- Implement feedback systems

**Topics:**
- Game loop concept
- Player feedback
- Difficulty and progression
- Polish and visual feedback

**Activities:**
- Analyze existing games
- Plan game mechanics
- Create design documents
- Implement visual feedback

**Project:** "Game Design Document"
- Written description of game concept
- List of mechanics and rules
- Sketch of game layout
- Difficulty progression plan

**Assessment:** Clear, well-thought-out game design

---

### Week 15-16 (or 15-20): Capstone Project

**Learning Objectives:**
- Integrate all semester concepts
- Create polished, playable game or interactive project
- Practice debugging and problem-solving
- Present work professionally

**Capstone Project Requirements:**

*Minimum Requirements:*
- 5+ sprites
- Score or progress tracking
- Win/lose or end condition
- At least 2 levels or scenes
- Proper use of loops, conditionals, and variables
- Polished appearance (colors, costumes, effects)
- Sound effects and/or music

*Enhanced Requirements (for advanced students):*
- 8+ sprites
- Multiple levels with increasing difficulty
- High score tracking
- Custom blocks for code organization
- Advanced interactions or mechanics

**Development Process:**
1. Week 1: Finalize design and create project plan
2. Week 2-3: Build game mechanics and core functionality
3. Week 4-5: Add polish, sounds, and refinements (if extended semester)
4. Final: Present and get peer feedback

**Assessment Criteria:**
- Functionality (does the game work?)
- Code organization and efficiency
- Creativity and polish
- Presentation quality
- Ability to explain design choices

---

## Assessment Methods

1. **Weekly Projects** - 50%
   - Application of learned concepts
   - Code quality and organization
   
2. **Quizzes and Concept Checks** - 20%
   - Understanding of programming concepts
   - Problem-solving ability
   
3. **Capstone Project** - 30%
   - Overall quality and functionality
   - Presentation and explanation
   - Creativity and polish

## Tips for Instructors

- Move at a steady pace to complete all units
- Focus on concepts over specific techniques
- Encourage students to remix and iterate
- Provide starter projects if students struggle
- Create a peer review system
- Celebrate progress and creativity
