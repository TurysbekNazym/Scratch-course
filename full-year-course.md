# Full Year Scratch Course (9 Months)

**Target Audience:** School children ages 8-12
**Duration:** 9 months (36 weeks)
**Frequency:** 1-2 hours per week
**Level:** Beginner → Intermediate → Advanced

---

## Course Overview

This comprehensive 9-month course takes students from Scratch beginners to confident programmers capable of creating complex interactive projects. Students progress through three phases: foundational concepts, intermediate programming techniques, and advanced project development.

---

## Learning Objectives

By the end of this course, students will be able to:
- Understand fundamental programming concepts (sequences, loops, conditionals)
- Create interactive projects with sprites and scripts
- Use variables and lists to manage data
- Build games and interactive stories
- Collaborate and share projects with others
- Debug and improve their programs

---

## Phase 1: Foundations (Months 1-3, 12 weeks)

### Month 1: Introduction to Scratch (Weeks 1-4)

**Week 1-2: Getting Started with Scratch**
- Learning Objectives:
  - Navigate the Scratch interface
  - Understand sprites, stages, and blocks
  - Create first simple script
- Topics:
  - Scratch account setup
  - Exploring the workspace
  - Motion blocks basics
  - Event blocks (when clicked, when key pressed)
- Project: Make a sprite move on the stage
- Assessment: Successfully move a sprite using motion blocks

**Week 3-4: Basic Motion and Events**
- Learning Objectives:
  - Use multiple event triggers
  - Chain blocks together
  - Understand block categories
- Topics:
  - Event-driven programming
  - Motion and appearance blocks
  - Sound blocks introduction
- Project: Interactive character (responds to key presses)
- Assessment: Create a sprite that moves in 4 directions with keyboard input

### Month 2: Sequences and Loops (Weeks 5-8)

**Week 5-6: Introduction to Loops**
- Learning Objectives:
  - Understand repetition concept
  - Use "repeat" and "forever" blocks
  - Recognize when loops are useful
- Topics:
  - Repeat block (fixed iterations)
  - Forever block (infinite loops)
  - Loop applications
- Project: Create animated sprite (walking, spinning)
- Assessment: Create animation using loops

**Week 7-8: Multiple Sprites and Interactions**
- Learning Objectives:
  - Work with multiple sprites
  - Control individual sprites
  - Create sprite interactions
- Topics:
  - Adding sprites
  - Sprite coordination
  - Touch/collision basics
- Project: Two-sprite interactive scene
- Assessment: Create scene with 2+ sprites that interact

### Month 3: Conditionals and Logic (Weeks 9-12)

**Week 9-10: Introduction to Conditionals**
- Learning Objectives:
  - Understand if/else logic
  - Use conditional blocks
  - Create decision-based programs
- Topics:
  - If block
  - If/else block
  - Comparison operators (>, <, =)
  - Boolean logic introduction
- Project: Simple game with scoring
- Assessment: Create program with conditional decision-making

**Week 11-12: Review Project - Phase 1**
- Learning Objectives:
  - Apply all Phase 1 concepts
  - Create complete interactive program
- Project: "Interactive Story" or "Simple Game"
  - 3+ sprites
  - Multiple scenes/screens
  - Scoring or point system
  - User interaction with keyboard/mouse
- Assessment: Comprehensive project review

---

## Phase 2: Intermediate Programming (Months 4-6, 12 weeks)

### Month 4: Variables and Data (Weeks 13-16)

**Week 13-14: Introduction to Variables**
- Learning Objectives:
  - Understand what variables are
  - Create and modify variables
  - Use variables to store data
- Topics:
  - Creating variables
  - Setting variable values
  - Using variables in scripts
  - Variable naming conventions
- Project: Score tracker game
- Assessment: Create program that tracks and displays variable values

**Week 15-16: Working with Multiple Variables**
- Learning Objectives:
  - Use multiple variables in one project
  - Coordinate variables
  - Create dynamic content based on variables
- Topics:
  - Multiple variable management
  - Variable operations
  - Displaying variables on stage
- Project: Quiz game or point-based game
- Assessment: Create game with scoring and lives system

### Month 5: Advanced Logic and Functions (Weeks 17-20)

**Week 17-18: Complex Conditionals and Operators**
- Learning Objectives:
  - Use AND/OR/NOT operators
  - Create complex decision logic
  - Debug conditional statements
- Topics:
  - Boolean operators
  - Nested conditionals
  - Logical reasoning
- Project: Advanced game with multiple conditions
- Assessment: Create program with complex if/else logic

**Week 19-20: Introduction to Custom Blocks (Advanced)**
- Learning Objectives:
  - Understand code organization
  - Create reusable code blocks
  - Use custom blocks effectively
- Topics:
  - Creating custom blocks (procedures/functions)
  - Block parameters
  - Code reusability
- Project: Project using 2-3 custom blocks
- Assessment: Create custom block and use in project

### Month 6: Game Design Fundamentals (Weeks 21-24)

**Week 21-22: Game Mechanics**
- Learning Objectives:
  - Understand game design principles
  - Implement win/lose conditions
  - Create game systems
- Topics:
  - Game loops
  - Scoring systems
  - Win/lose conditions
  - Game state management
- Project: Design simple game prototype
- Assessment: Document game design and mechanics

**Week 23-24: Mid-Year Project**
- Learning Objectives:
  - Integrate all Phase 2 concepts
  - Create polished game or interactive project
- Project: Complete game or interactive story
  - 5+ sprites
  - Score/lives system
  - Multiple levels or scenes
  - Win/lose conditions
  - Custom blocks for organization
- Assessment: Project presentation and playability testing

---

## Phase 3: Advanced Projects (Months 7-9, 12 weeks)

### Month 7: Lists and Data Management (Weeks 25-28)

**Week 25-26: Introduction to Lists**
- Learning Objectives:
  - Understand lists and arrays
  - Create and manipulate lists
  - Use lists for data storage
- Topics:
  - Creating lists
  - Adding/removing items
  - Accessing list items
  - Looping through lists
- Project: High score tracker or inventory system
- Assessment: Create program using lists

**Week 27-28: Advanced List Operations**
- Learning Objectives:
  - Use lists in complex programs
  - Implement list-based systems
- Topics:
  - List operations
  - Nested data structures
  - List applications in games
- Project: Game with inventory or multiple levels stored in list
- Assessment: Complex data management project

### Month 8: Cloning and Advanced Techniques (Weeks 29-32)

**Week 29-30: Cloning Sprites**
- Learning Objectives:
  - Understand sprite cloning
  - Create dynamic sprite systems
  - Manage cloned sprites
- Topics:
  - Clone block
  - Managing clones
  - Clone applications (bullets, enemies, collectibles)
- Project: Game with cloned enemies or projectiles
- Assessment: Create working clone system

**Week 31-32: Broadcasting and Advanced Coordination**
- Learning Objectives:
  - Use broadcast messages
  - Coordinate complex sprite behavior
  - Create sophisticated interactions
- Topics:
  - Broadcast and receive
  - Message-driven programming
  - Sprite coordination patterns
- Project: Multi-sprite game with complex interactions
- Assessment: Project with multiple broadcast messages

### Month 9: Capstone Project (Weeks 33-36)

**Week 33-34: Project Planning and Design**
- Learning Objectives:
  - Plan complex project
  - Design game mechanics
  - Plan code structure
- Activities:
  - Define project scope
  - Create design document
  - Plan sprite behaviors
  - Outline variables and lists needed
- Deliverable: Design document

**Week 35-36: Capstone Project Development and Presentation**
- Learning Objectives:
  - Create comprehensive final project
  - Practice problem-solving
  - Present work professionally
- Final Project Requirements:
  - 8+ sprites
  - Multiple scenes/levels
  - Complete scoring/lives system
  - Use of variables, lists, and custom blocks
  - Polished user interface
  - Sound and visual effects
  - Appropriate difficulty progression

- Assessment:
  - Project code quality and organization
  - Functionality and gameplay
  - Creativity and originality
  - Presentation skills
  - Peer feedback and reflection

---

## Assessment Methods

1. **Weekly Projects** - Hands-on application of concepts (40%)
2. **Quizzes** - Understanding of programming concepts (20%)
3. **Code Quality** - Organization, comments, efficiency (20%)
4. **Presentation** - Ability to explain and demo projects (20%)
